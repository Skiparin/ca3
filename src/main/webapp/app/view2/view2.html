Hvem gjorde hvad:

Vi har allesammen arbejdet med alle de forskellige værktøjer som f.eks. API,
HTML, Angular og databaser. Vi føler alle har gjort et stykke arbejde for, at få
dette projekt igennem og hjulpet hinanden med de ting de har haft problemer med
og fået folk igennem deres svage sider. Derfor mener vi alle har fortjent fuldt studypoint.
Måden vi fik delt arbejdet ud på, var at vi spurgte hvad folk ville arbejde med.
Efter noget tid gjorde vi, at man fik prøvet noget andet. Hvis man fik problemer,
spurgte man i gruppen og så om en fra gruppen, havde prøvet at arbejde med problemet
der var opstået. På denne måde var vores arbejde ”individuelt”, men vi sad stadig
som en gruppe, og fik hinanden igennem de punkter vi fandt svære.
Med git lavede vi en branch for hver ”emne” som vi arbejdede med. Vi mergede det
så ind i masterbranchen, når vi følte vores del af koden virkede.
Når koden var merget ind i masterbranchen, blev den branch der var arbejdet på slettet.
Der blev derefter oprettet en ny branch man kunne arbejde i. På denne måde endte
vi ikke med at folk havde pushet noget op på master branchen, og gjorde at nogle
af de andre ting i koden stoppede med at virke, og vi fik merge conflicts.

Company info:

Vi har lavet et UI til users, så de kan finde det company de leder efter.
Man indtaster sit søgord og trykker "slå op", man får så et table med alt dataen
på det søgte company. Nederst kan man vælge productunit og få dataen
på den valgte unit. Der er også sat en tilbage knap ind, så man kan komme
tilbage til company info.

Exchange Rate:

På denne side kan man se alle rates, på de currencies vi får fra API'en.
Filen "XmlReaderDemo" i Source Packages.utils skal køres for at få dataen i databasen
For at vi kan få dem udskrevet og bruge calculatoren på siden, skal "get rates"
trykkes på først. Derefter kan man vælge hvilken currency man vil veksle fra,
og hvilken currency man vil veksle til og hvor mange penge man vil veksle.
Derefter klikker man calculate, og får hvor mange man får af den vekslede valuta.

All users:

All users kan kun ses af admin og giver mulighed for, at se alle users som kan
bruges på siden og ligger i databasen. Admins kan også slette brugerne fra siden/databasen.
Efter useren er deletet vil personen forsvinde fra listen som adminen kan se.

Create user:

Denne metode opretter en user og kigger på, om en user med det username allerede
er i databasen. Hvis dette er tilfældet vil brugeren ikke blive oprettet.
Hvis der derimod ikke eksistere en bruger i databasen med det valgte username,
vil useren blive smidt i databasen med "user" som "role". Derefter vil useren
være mulig at bruge til at logge ind med.

Test

Vi prøvede at lave rest test men de gav fejl. Alle vores JUNIT tests virker
og giver rigtigt
Createuser:
Denne test tester vores metode createUser() i vores UserFacade. Metoden tager imod
username og password, og kigger så på om usernamet er i brug. Hvis usernamet
ikke er i brug, vil metoden oprette et user objekt med dataen og give den en ”role”.
Denne role vil altid være ”User”. Efter metoden har kørt, vil den returnere
user objektet med user rolen.
Deleteuser:
Vores metode deleteUserById() fjerner en user fra vores database med det givne ID.
I denne opgave er username sat som vores ID, så metoden gør det ud fra username.
Metoden leder i databasen efter en user med det valgte username, og fjerner derefter
den user fra databasen. Metoden returnerer derefter den slettede user.
GetAllUsers:
Metoden getAllUsers() henter alle users i databasen og returnerer dem som en liste
af user objects. Vi tester ved at oprette en user med vores createUser(), hvor vi
derefter henter listen med users og ser om useren vi har oprettet ligger i listen.
